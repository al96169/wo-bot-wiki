# wo-bot 项目总览

> 通用开源家庭机器人控制平台

## 一、项目简介

wo-bot 是一个**机器人远程控制与智能交互平台**。用户通过浏览器或手机 App，在任何网络环境下连接和控制机器人，实现实时视频监控、语音喊话、云台操控、自主运动等功能。

| 项目 | 内容 |
|------|------|
| GitHub 主页 | https://github.com/al96169 |
| 作者 | Antonio Leung |
| 开源协议 | MIT |
| 包名前缀 | com.antonioleung.wobot |
| 硬件平台 | NVIDIA Jetson（Orin NX/AGX） + 小车底盘 + 摄像头 + 云台 + 麦克风/扬声器 |
| 目标场景 | 家庭安防巡检、远程看护、娱乐互动、教育展示 |

## 七、安全设计

跨网络远程控制涉及设备访问权限，信令服务器通过**四层安全机制**确保只有设备主人能连接设备。

### 1. 客户端认证（JWT 令牌）

浏览器连接信令服务器时必须携带有效 JWT 令牌：

```
wss://signal.wo-bot.com/ws?role=client&token=<JWT>&robotId=<设备ID>
```

- JWT 包含 `userId`（用户身份），使用 HS256 算法签名，有过期时间
- 信令服务器在本地完成签名验证，不依赖外部服务
- 没有有效令牌的连接会被直接关闭（WebSocket close code 1008）

### 2. 设备归属校验（核心防线）

JWT 验证通过只说明"你是某个用户"，但**不代表有权访问这个设备**。信令服务器额外调用帐号服务器 API 查询设备归属：

```
GET /api/devices/{robotId}/owner  →  返回设备绑定的 userId
```

- 比对 JWT 中的 `userId` 与设备实际 `owner`，两者一致才允许连接
- 即使攻击者拿到有效 JWT，也无法连接不属于他的设备
- 归属校验结果有 5 分钟缓存，减少对帐号服务器的调用压力

### 3. 机器人认证（HMAC-SHA256 + 时间戳）

机器人端连接信令服务器时使用对称密钥签名：

```
signature = HMAC-SHA256(ROBOT_SECRET, "deviceId:timestamp")
```

- 信令服务器用同样的 `ROBOT_SECRET` 重新计算并比对（constant-time 比较防时序攻击）
- 时间戳窗口限制在 5 分钟内，超过则拒绝，防止重放攻击
- `ROBOT_SECRET` 只存在于机器人和信令服务器中，不经过客户端

### 4. 基础防护

- **限流**：单 IP 每秒最多 3 次连接尝试，防止暴力探测
- **TURN 短期凭证**：WebRTC 中继使用 24 小时短期凭证（HMAC-SHA1），不可伪造，过期后自动失效
- **WebRTC 加密**：数据通道和视频流在传输层强制加密（SRTP / DTLS）

### 攻击场景分析

| 攻击方式 | 能否成功 | 拦截层 |
|---------|---------|--------|
| 无令牌直接连接 | ✗ | JWT 验证 |
| 拿到他人 JWT 连接非本人设备 | ✗ | 设备归属校验 |
| 伪造机器人身份 | ✗ | HMAC 签名验证 |
| 重放机器人签名 | ✗ | 时间戳窗口 |
| 暴力探测设备 | ✗ | 限流 |
| 拦截通信数据 | ✗ | WebRTC SRTP/DTLS 加密 |

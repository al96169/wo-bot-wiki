# 省电策略与第三方服务接入方案

> 目标：设计一套对第三方开发者友好的省电接入机制，让服务开发者自然遵守省电协议，而非被平台硬编码限制。

---

## 一、现状：当前省电策略实现

### 1.1 架构概览

```
 Rosmaster 串口          SystemCollector              PowerPolicy              MessageHandler              WebSocket/WebRTC            前端 Vue
 (硬件电池电压)  ──→  (电压→百分比+时长估算)  ──→  (模式决策引擎)  ──→  (消息路由+限流)  ──→  (双通道推送)  ──→  (Store+UI)
```

### 1.2 各层职责

| 层 | 文件 | 职责 |
|----|------|------|
| 采集层 | `collector.py` | 从 Rosmaster 串口读取 12V 锂电池真实电压，线性映射为百分比，估算剩余时长 |
| 引擎层 | `power_policy.py` | 状态机：`normal/eco` 两模式、`manual_override` 豁免、`simulated_battery` 调试 |
| 路由层 | `message_handler.py` | 5 个 handler 处理省电相关消息，硬编码限流点（dance 禁止、音量上限 50%） |
| 推送层 | `main.py` | 模式变更时 WebSocket 广播 `service_message` + `power_policy_status` |
| 前端 | `ConfigView.vue` + `robot.ts` | 模式展示、阀值配置、模拟电量调试 |

### 1.3 PowerPolicy 状态机

```
                   电量 ≤ threshold(30%) + 无 manual_override
  ┌─────────┐  ───────────────────────────────────────────→  ┌─────────┐
  │ NORMAL  │                                                │   ECO   │
  └─────────┘  ←───────────────────────────────────────────  └─────────┘
                   电量 ≥ 80%（硬编码，不可配置）                 │
                   或 用户手动 toggle                          │
                                                             │
  手动切回 NORMAL → 设置 manual_override（本次会话不再自动进入 ECO）  │
  进入 ECO → 清除 manual_override                               │
```

### 1.4 省电限流点（当前硬编码在 MessageHandler）

| 服务 | 限流方式 | 触发位置 |
|------|---------|---------|
| 跳舞 | 拒绝非只读命令，返回 403 | `_handle_dance()` L268 |
| 音乐音量 | 强制上限 50% | `_handle_music_volume()` L976 |

### 1.5 关键问题

- **限流点硬编码**：每个服务需要在 `message_handler.py` 里手写 `if power_policy.is_eco`，无法扩展
- **第三方服务无接入点**：子进程服务完全不知道省电状态，进程内 ExtensionModule 无省电钩子
- **二元开关太粗糙**：只有 `normal/eco` 两档，无法做渐进降级
- **开发者无感知**：服务被限制时不被告知原因和当前电量

## 二、推荐方案：多级省电 + 服务优先级

### 2.1 从二元到多级

借鉴 Android Battery Saver 的分级设计，将 `normal/eco` 升级为四级：

```python
# power_policy.py — 多级省电定义

class PowerLevel(Enum):
    NORMAL = "normal"           # > 50%: 无限制
    ECO_LOW = "eco_low"         # 30-50%: 轻度限制
    ECO_MID = "eco_mid"         # 15-30%: 中度限制
    ECO_CRITICAL = "eco_critical"  # < 15%: 仅保留运动+通信

POWER_LEVEL_CONFIG = {
    PowerLevel.NORMAL: {
        "threshold": 50,  # 电量 ≤ 50% 进入下一级
        "max_volume": 100,
        "allow_dance": True,
        "allow_camera": True,
        "default_fps": 30,
    },
    PowerLevel.ECO_LOW: {
        "threshold": 30,
        "max_volume": 70,
        "allow_dance": True,
        "allow_camera": True,
        "default_fps": 15,
    },
    PowerLevel.ECO_MID: {
        "threshold": 15,
        "max_volume": 50,
        "allow_dance": False,
        "allow_camera": True,
        "default_fps": 5,
    },
    PowerLevel.ECO_CRITICAL: {
        "threshold": 0,
        "max_volume": 30,
        "allow_dance": False,
        "allow_camera": False,
        "default_fps": 0,
    },
}
```

### 2.2 服务优先级桶

借鉴 Android 9+ App Standby Buckets，每个服务注册时声明优先级：

| 优先级 | 含义 | 省电行为 | 示例服务 |
|--------|------|---------|---------|
| `critical` | 核心功能，不可中断 | 始终运行，仅在 critical 时降频 | 运动控制、遥控通信 |
| `high` | 重要功能 | 轻度省电降频，中度省电降质 | 摄像头推流、SLAM |
| `normal` | 普通功能 | 轻度省电限参，中度省电限功能 | 音乐播放、语音助手 |
| `low` | 非必需功能 | 轻度省电建议降级，中度省电直接暂停 | 跳舞、激光雷达扫描 |
| `idle` | 纯后台/可随时停止 | 省电时立即暂停 | 日志采集、软件更新检查 |

```python
# service_manager.py — SERVICE_DEFINITIONS 扩展

SERVICE_DEFINITIONS = {
    "motion_control": {
        "priority": "critical",
        "auto_start": True,
        "in_process": True,
    },
    "remote_control": {
        "priority": "critical",
        "in_process": True,
    },
    "camera_stream": {
        "priority": "high",
        "power_behavior": {
            "eco_low":     {"fps": 15},
            "eco_mid":     {"fps": 5, "resolution": "480p"},
            "eco_critical": "disabled",
        },
    },
    "music_player": {
        "priority": "normal",
        "power_behavior": {
            "eco_low":     {"volume": 70},
            "eco_mid":     {"volume": 50},
            "eco_critical": {"volume": 30},
        },
    },
    "dance": {
        "priority": "low",
        "power_behavior": {
            "eco_low":     "enabled",
            "eco_mid":     "disabled",
            "eco_critical": "disabled",
        },
    },
}
```

## 三、第三方服务三层接入机制

```
┌─────────────────────────────────────────────────────┐
│  Layer 1: 声明式省电配置（零代码，覆盖 80% 场景）      │
│  Layer 2: 省电状态广播（子进程自动收到电量推送）       │
│  Layer 3: 生命周期钩子（进程内服务自主降级）           │
└─────────────────────────────────────────────────────┘
```

### 3.1 Layer 1：声明式 — 注册时填配置

开发者只需在 `SERVICE_DEFINITIONS` 里声明 `priority` 和 `power_behavior`，平台在 MessageHandler 统一拦截：

```python
# message_handler.py — 通用省电拦截器

async def _check_power_policy(self, service_id: str, data: dict) -> dict | None:
    """统一的省电检查。返回 None 放行，返回 dict 拦截。"""
    if not self.power_policy or self.power_policy.level == PowerLevel.NORMAL:
        return None

    svc_def = SERVICE_DEFINITIONS.get(service_id, {})
    behavior = svc_def.get("power_behavior", {})

    current_level = self.power_policy.level.value
    policy = behavior.get(current_level, None)

    if policy is None:
        return None  # 未声明策略则放行

    if policy == "disabled":
        return {"type": "error", "data": {"code": 403, "message": "省电模式下该功能不可用"}}

    if isinstance(policy, dict):
        # 强制修改限制字段（如音量上限、帧率上限）
        for field, max_val in policy.items():
            if field in data and isinstance(data[field], (int, float)) and data[field] > max_val:
                data = dict(data)
                data[field] = max_val
        return None  # 参数已被限制，继续执行

    return None
```

### 3.2 Layer 2：子进程广播 — 自动收到省电状态

模式变更时，平台向所有子进程 stdin 推送 `power_policy_status` 广播消息：

```python
# service_manager.py — 省电状态广播

async def broadcast_power_policy_status(self, status: dict):
    """省电模式变更时，向所有子进程推送状态"""
    message = json.dumps({
        "type": "power_policy_status",
        "data": status,
    }) + "\n"
    for proc_info in self._subprocesses.values():
        if proc_info.get("proc") and proc_info["proc"].stdin:
            try:
                proc_info["proc"].stdin.write(message.encode())
                await proc_info["proc"].stdin.drain()
            except Exception:
                pass
```

子进程端只需在 `_reader_loop` 里处理广播消息：

```python
# 第三方子进程服务模板
async def _reader_loop():
    reader = asyncio.StreamReader()
    await loop.connect_read_pipe(lambda: asyncio.StreamReaderProtocol(reader), sys.stdin)

    power_mode = "normal"
    battery_level = 100

    while True:
        line = await reader.readline()
        if not line:
            break
        msg = json.loads(line.decode("utf-8").strip())

        # 省电状态广播（无 id 字段，不是请求-响应）
        if msg.get("type") == "power_policy_status":
            power_mode = msg["data"]["level"]
            battery_level = msg["data"].get("battery_level", battery_level)
            continue

        # 正常请求处理（可自主判断省电状态）
        result = await handle_command(msg["cmd"], msg.get("params", {}), power_mode)
        result["id"] = msg["id"]
        sys.stdout.write(json.dumps(result) + "\n")
        sys.stdout.flush()
```

### 3.3 Layer 3：进程内钩子 — 覆盖一个方法

在 ExtensionModule 基类加入省电生命周期钩子：

```python
# modules/extension/base.py — 基类增强

class ExtensionModule(ABC):
    def __init__(self, module_id: str, config=None, logger=None):
        # ... 现有字段 ...
        self._power_mode = "normal"
        self._battery_level = 100

    @property
    def power_mode(self) -> str:
        return self._power_mode

    @property
    def is_eco(self) -> bool:
        return self._power_mode != "normal"

    async def on_power_mode_changed(self, from_mode: str, to_mode: str):
        """省电模式变更回调。子类覆盖以实现优雅降级/恢复。"""
        self._power_mode = to_mode

    def _update_power_state(self, mode: str, battery_level: int):
        """由平台调用，更新省电状态"""
        self._power_mode = mode
        self._battery_level = battery_level
```

第三方服务覆盖回调实现自主降级：

```python
class MyLidarService(ExtensionModule):
    async def on_power_mode_changed(self, from_mode: str, to_mode: str):
        await super().on_power_mode_changed(from_mode, to_mode)
        if to_mode.startswith("eco"):
            await self.lidar.set_frequency(5)   # 降频至 5Hz
        else:
            await self.lidar.set_frequency(20)  # 恢复正常
```

## 四、安卓厂商借鉴总结

| Android 机制 | wo-bot 对应方案 | 优先级 |
|---|---|---|
| **多级省电**（Battery Saver 分级） | `PowerLevel` 四级 + `POWER_LEVEL_CONFIG` 策略表 | 高 |
| **App Standby 桶**（优先级分组） | `SERVICE_PRIORITY` 五级 + 声明式 `power_behavior` | 高 |
| **WorkManager**（系统统一调度） | `scheduler.submit(task, min_battery, eco_window)` API | 中 |
| **维护窗口**（Doze 窗口集中执行） | 省电时对 low 优先级服务做周期唤醒而非持续运行 | 中 |
| **SIGSTOP 挂起**（OPPO 进程冻结） | 省电时挂起子进程而非杀死，恢复时秒唤醒 | 低 |
| **用户可感知的细分控制**（三星自适应） | 前端展示各服务省电状态，支持用户手动放行 | 低 |

## 七、向后兼容

- 未声明 `priority` 的服务默认 `priority: "normal"`，行为与当前一致
- 未声明 `power_behavior` 的服务不受省电限制（等同于 `always_on`）
- `get_status()` 返回的 `mode` 字段保持 `"normal"` / `"eco"` 字符串，新增 `level` 字段为 `"normal" | "eco_low" | "eco_mid" | "eco_critical"`
